Accessibility Statement

Last Updated: October 31, 2025

Our Commitment

WFY Solutions is committed to ensuring that BARS AI is accessible to all users, including those with disabilities. We believe that everyone should have equal access to business intelligence tools, regardless of ability.

Conformance Status

We aim to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA. These guidelines explain how to make web content more accessible for people with disabilities.

Current Status: We are actively working toward full WCAG 2.1 Level AA conformance. Some features may not yet be fully compliant, and we appreciate your patience as we continue to improve.

Accessibility Features

Keyboard Navigation

  • All interactive elements can be accessed using keyboard only
  • Logical tab order throughout the interface
  • Visible focus indicators on all focusable elements
  • Skip links to bypass repetitive navigation

Screen Reader Compatibility

  • Semantic HTML markup for proper content structure
  • ARIA labels and landmarks for navigation
  • Alt text for all meaningful images
  • Proper heading hierarchy (H1-H6)
  • Compatible with JAWS, NVDA, VoiceOver, and TalkBack

Visual Design

  • High contrast ratios (minimum 4.5:1 for normal text)
  • Resizable text up to 200% without loss of functionality
  • Clear visual indicators for links and buttons
  • No reliance on color alone to convey information
  • Readable font sizes and line spacing

Forms and Input

  • Clear labels for all form fields
  • Error messages that explain how to correct problems
  • Descriptive button text
  • Inline validation feedback

Content Structure

  • Consistent navigation across pages
  • Clear page titles
  • Logical content flow
  • Descriptive link text (no "click here")

Known Limitations

Despite our efforts, some accessibility issues may exist:

  • Some complex data visualizations may not be fully accessible to screen readers
  • PDF reports generated may require additional accessibility optimization
  • Third-party integrations (QuickBooks, Sage) may have their own accessibility limitations

Accessibility Testing

We regularly test BARS AI for accessibility using:

  • Automated testing tools (WAVE, aXe)
  • Manual keyboard navigation testing
  • Screen reader testing with real users
  • Color contrast verification
  • User feedback and reports

Third-Party Content

We integrate with third-party services (QuickBooks, Sage, Stripe) that have their own accessibility practices. While we advocate for accessibility, we cannot guarantee the accessibility of these external services.

Alternative Formats

If you need content in an alternative format:

  • Reports can be exported to multiple formats (PDF, Excel, CSV)
  • We can provide documentation in accessible formats upon request
  • Contact us for assistance with specific accessibility needs
